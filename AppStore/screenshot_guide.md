# THDatabase 截图制作指南

> 用于 App Store 上架的截图制作参考

---

## 截图规格要求

### 必填设备尺寸

| 设备 | 分辨率（px） | 数量 |
|------|------------|------|
| iPhone 6.7"（iPhone 15 Pro Max） | 1290 × 2796 | 5 张 |
| iPhone 6.5"（iPhone 14 Pro Max） | 1284 × 2778 | 5 张 |
| iPad Pro 12.9"（第 6 代） | 2048 × 2732 | 5 张（如支持 iPad） |

### 格式要求

- 文件格式：PNG 或 JPEG
- 色彩空间：RGB
- 不允许透明背景
- 每张截图最大 500 MB

---

## 推荐截图内容（5 张）

### 截图 1：连接列表页

**展示内容：**
- 应用主界面，显示已保存的数据库连接列表
- 至少显示 2-3 个连接（SQLite + MySQL）
- 右上角的"+"按钮清晰可见

**文案标注建议：**
```
管理多个数据库连接
支持 SQLite 和 MySQL
```

**制作要点：**
- 使用真实但脱敏的连接名称（如"开发环境 MySQL"、"本地测试 SQLite"）
- 确保 UI 元素清晰，字体可读
- 展示应用的整洁界面设计

---

### 截图 2：新建连接页

**展示内容：**
- 新建/编辑连接的表单界面
- 填写 MySQL 连接信息（主机、端口、用户名、数据库名）
- 数据库类型选择器显示"MySQL"

**文案标注建议：**
```
简洁的连接配置
快速连接到远程数据库
```

**制作要点：**
- 使用示例数据（如主机：192.168.1.100，端口：3306）
- 不要显示真实的密码或敏感信息
- 展示表单的清晰布局

---

### 截图 3：SQL 编辑器页（带侧边栏）

**展示内容：**
- 数据库工作区界面
- 左侧侧边栏显示数据库列表和表列表
- 中间 SQL 编辑器输入框，输入一条 SELECT 查询
- 例如：`SELECT * FROM users LIMIT 10;`

**文案标注建议：**
```
强大的 SQL 编辑器
侧边栏快速浏览 Schema
```

**制作要点：**
- 侧边栏展开，显示至少 3-5 个表名
- SQL 语句使用语法高亮（如果有）
- 展示专业的开发工具界面

---

### 截图 4：查询结果页

**展示内容：**
- 查询结果以表格形式展示
- 显示列名和至少 5-10 行数据
- 数据清晰可读，支持横向滚动

**文案标注建议：**
```
清晰的结果展示
支持横向滚动查看更多列
```

**制作要点：**
- 使用脱敏的示例数据（如用户名：user001、user002）
- 确保表格布局整齐
- 展示多列数据（至少 4-5 列）

---

### 截图 5：写操作确认弹窗

**展示内容：**
- SQL 编辑器中输入一条 DELETE 或 UPDATE 语句
- 弹出确认对话框，提示用户确认操作
- 对话框显示警告图标和"确认"/"取消"按钮

**文案标注建议：**
```
安全保护机制
写操作前需确认，防止误操作
```

**制作要点：**
- 使用 DELETE 或 DROP 等高危操作作为示例
- 确认对话框清晰可见
- 展示应用的安全特性

## 截图制作工具

### 方法 1：真机截图（推荐）

1. 在真机或模拟器上运行应用
2. 准备好测试数据（连接配置、示例数据库）
3. 按照上述顺序截取 5 张截图
4. 使用 macOS 自带的"预览"或 Xcode 的截图工具

### 方法 2：设计工具美化

如果需要添加文案标注或美化：
- 使用 Figma、Sketch 或 Photoshop
- 在截图上添加文字说明
- 保持 App Store 风格（简洁、专业）

---

## 注意事项

### ✅ 应该做的

- 使用真实的应用界面截图
- 数据脱敏（不显示真实密码、敏感信息）
- 展示应用的核心功能
- 保持截图清晰、高分辨率
- 按照功能流程排序（连接 → 编辑 → 查询 → 结果 → 安全）

### ❌ 不应该做的

- 不要使用模拟或 PS 的假界面
- 不要显示真实的生产数据库信息
- 不要过度美化，导致与实际应用不符
- 不要包含其他应用的界面或品牌
- 不要使用低分辨率或模糊的图片
